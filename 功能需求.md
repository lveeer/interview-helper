# 面向求职者的智能面试提升系统功能需求文档

## 1. 引言

### 1.1 编写目的
本文档旨在明确“面向求职者的智能面试提升系统”的功能需求、性能要求及系统约束，为系统的设计与实现提供依据。

### 1.2 项目背景
随着高校毕业生规模增加，求职竞争激烈。传统面试准备方式缺乏针对性指导、反馈滞后且真人模拟成本高昂。本项目利用生成式大语言模型（LLM）及 RAG 技术，构建一套集简历解析、岗位匹配、AI模拟面试及知识强化于一体的智能系统，以提升求职者的面试竞争力。

### 1.3 适用范围
本系统主要面向高校求职毕业生，提供从简历优化到模拟实战的全流程辅助服务。

| 角色名称        | 角色描述                                       | 核心目标                                             |
| :-------------- | :--------------------------------------------- | :--------------------------------------------------- |
| **求职者/用户** | 系统的主要使用者，正在准备面试的学生或求职人员 | 优化简历、获取岗位匹配度、进行模拟面试练习、查漏补缺 |

## 4. 非功能性需求

### 4.1 性能需求
*   **响应速度**：系统后端采用 FastAPI，应保证简历解析、问题生成及追问的响应时间在用户可接受范围内（一般不超过 3-5 秒）。
*   **并发处理**：支持多用户同时进行模拟面试，WebSocket 长连接保持稳定。

### 4.2 安全性需求
*   **隐私保护**：用户简历及导入的私有题库包含敏感信息，系统必须采用隐私脱敏技术处理，并确保数据传输与存储的安全。
*   **数据隔离**：不同用户的数据（如个人知识库）必须严格隔离，确保私有数据不被其他用户检索到。

### 4.3 可用性需求
*   **界面友好**：前端基于 Vue3 + ElementPlus 构建，界面需美观、易用，交互流程顺畅。
*   **容错性**：针对 LLM 可能产生的“幻觉”现象，系统需具备提示词优化策略，确保对话逻辑性和拟人化程度。

### 4.4 数据存储需求
*   **混合存储**：使用 PostgreSQL 存储结构化数据，使用 pgvector 插件存储向量数据，确保向量检索的准确性与效率。

## 5. 系统数据流概要

1.  **输入阶段**：用户上传简历（文件） + 输入目标JD（文本） + 导入特定文档（文件）。
2.  **处理阶段**：
    *   Unstructured.io 解析简历 -> LLM 提取结构化信息。
    *   LLM 计算 JD 匹配度。
    *   RAG 技术向量化导入文档 -> 存入向量库。
3.  **交互阶段**：
    *   用户进入面试房间。
    *   LLM 生成问题 -> 用户回答（语音/文字）。
    *   LLM 结合上下文（简历+JD+私有库）生成追问。
4.  **输出阶段**：LLM 评分 + 生成 Markdown 报告 -> 系统关联推荐学习资料 -> 前端可视化展示。

## 6. 附录：关键技术栈
*   **前端**：Vue3 + ElementPlus + Pinia
*   **后端**：FastAPI + Pydantic
*   **数据库**：PostgreSQL + pgvector
*   **AI 技术**：大语言模型 (LLM), RAG (检索增强生成), Unstructured.io
